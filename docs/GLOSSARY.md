# GLOSSARY.md

Domain vocabulary for CashFB. When naming something in code, check here first for an existing term.

---

## A

**Ad placement key**
Stable string identifying where an ad appears (`home_top_banner`, `timer_bottom_banner`, `redeem_code_bottom_banner`, `custom_room_bottom_banner`, `result_middle_banner`, `timer_top_banner`). Stored on `ads_config.placementKey`. Stays the same across ad networks so swapping networks doesn't require app changes.

**Admin panel**
Flutter Web app used by the operator team. Talks to the same backend as the consumer app.

**App config**
Single-document collection (`app_config`, keyed by `'default'`) holding runtime-tunable values: base rate, coin costs, signup bonus, blocked states, feature flags.

**Audit log**
Row in `audit_logs`. One per admin write action. Captures `actor`, `action`, `resource`, `before`, `after`, `ip`, `userAgent`.

---

## B

**BGMI**
Battlegrounds Mobile India. One of two custom-room game types (along with Free Fire).

**Batch (redeem code batch)**
A set of gift codes uploaded together via CSV. Tracked in `redeem_code_batches`. Has a supplier, denomination, and invoice URL for audit.

**Base rate (prize pool)**
Paise per vote multiplier used in the daily prize pool calculation. Default 100 (₹1). Stored in `app_config.baseRatePerVote`.

**Bull Board**
Web UI for BullMQ queues. Mounted at `/admin/queues` for SUPER_ADMIN.

---

## C

**CashFB**
The product name.

**CMS content**
HTML content managed via admin panel. Keys: `TERMS`, `HOW_DISTRIBUTE`, `FAQ`, `PRIVACY`, `GRIEVANCE`. Served at `GET /cms/:key`.

**Coin**
In-app currency. Not money. Not withdrawable. Used only to gate voting. Earned by completing posts (1 coin each) and granted on signup (3 coins). Spent on voting (3 coins per vote).

**Coin transaction**
Row in `coin_transactions`. Every coin movement has one. Type is `SIGNUP_BONUS`, `POST_REWARD`, `VOTE_SPEND`, `ADMIN_CREDIT`, `ADMIN_DEBIT`, or `REFUND`.

**Copy (gift code copy)**
The action of claiming a published gift code. Atomic, first-come-first-serve. Only the first user's `findOneAndUpdate` succeeds.

**Counter**
Row in `counters`. Monotonic integer sequence keyed by purpose (e.g. `invoice:2026-27`). Atomic via `findOneAndUpdate({key}, {$inc: {value: 1}})`.

**Custom room**
A scheduled BGMI or Free Fire match. Admin creates it with encrypted `roomId` and `roomPwd`. Credentials reveal at `visibleFromAt`. Results publishable 30 min after start (`resultEnabledAt`).

---

## D

**DAU**
Daily active users. Counted via distinct `login_sessions.userId` in the last 24 h.

**Day key**
`'YYYY-MM-DD'` string in IST. Used for `votes.dayKey`, `posts.dayKey`, `post_completions.dayKey`, `prize_pools.dayKey`, `custom_rooms.dayKey`. Cheaper to index and compare than `Date`.

**DEK**
Data Encryption Key. Short-lived AES-256 key generated by AWS KMS for a single encrypt operation. Envelope scheme: DEK encrypts the plaintext, KMS CMK encrypts the DEK.

**Device fingerprint**
Hash of Android ID + build props + install UUID, generated client-side. Stored on `device_fingerprints`. Used for anti-fraud and refresh-token device binding.

**DLT**
Distributed Ledger Technology, India's SMS template registration system. MSG91 templates must be DLT-registered before sending OTPs to Indian numbers.

**DPDP Act**
Digital Personal Data Protection Act 2023. India's primary data protection law. DPDPR 2025 (rules) notified Nov 2025; operational compliance by May 2027.

**Donor**
Anyone who donates via Razorpay on the home-screen sponsor section. Stored in `donations`. Cumulative totals power the Top Donor and Top 50 leaderboards.

---

## F

**FCFS**
First-come-first-serve. The gift code distribution model.

**FCM**
Firebase Cloud Messaging. Push notification provider.

**Feature flag**
Boolean or structured value in `app_config.featureFlags` controlling whether a feature is enabled. Used to keep PROGA-risky features dormant until legal sign-off.

**Free Fire (FF)**
Garena Free Fire. The second custom-room game type.

---

## G

**Gift code**
A Google Play gift card code. Pre-purchased in bulk from an authorised B2B reseller. ₹50 denomination. Stored encrypted in `redeem_codes`.

**GSTIN**
15-character Goods and Services Tax Identification Number. Required on every GST-compliant invoice.

**Gross-up**
TDS strategy for in-kind prizes. Instead of deducting tax from the face value, reduce the awarded amount so that `faceValue - TDS = prizeValue`.

---

## H

**Home feed**
Aggregated payload returned by `GET /home`. Includes top donor, 3 sponsors, today's posts, today's pool, tier context. Cached in Redis 30s.

---

## I

**IMEI hash**
Hash of the device IMEI, stored on `device_fingerprints.imeiHash`. Never store raw IMEI.

**Invoice number**
Sequential per financial year, format `CF/YYYY-YY/NNNNNN`. Generated via atomic `$inc` on the `counters` collection keyed by `invoice:<FY>`.

**IST**
Indian Standard Time (UTC+5:30). Default timezone for all scheduling and day keys.

---

## J

**JTI**
JWT ID claim. Unique per token. Used as the session primary key in `login_sessions`.

---

## K

**KMS**
AWS Key Management Service. CashFB uses a customer-managed CMK in ap-south-1 for envelope encryption of gift codes, room passwords, and PANs.

**KYC threshold**
Prize value (paise) above which PAN collection is required before payout. Stored in `app_config.kycThresholdAmount`. Default 10000 (₹100).

---

## M

**MAU**
Monthly active users. Distinct `login_sessions.userId` in last 30 days.

**MSG91**
Primary SMS OTP provider for Indian numbers. DLT-registered templates required.

**Multiplier**
Tier-based winnings multiplier. Public 1x, Pro 5x, Pro Max 10x. Stored on `PrizePoolWinner.multiplier` per prize row.

---

## O

**OTP**
One-Time Password. 6-digit numeric code sent via SMS or email for signup/login. Hashed on storage, 5-min TTL, 5 max attempts.

---

## P

**PAN**
Permanent Account Number. 10-character Indian tax identifier. Collected lazily before first prize payout above `kycThresholdAmount`. Encrypted at rest. `panLast4` stored in plaintext for display only.

**Pool (prize pool)**
Daily prize amount, computed from yesterday's vote count x base rate. Split 70/30 between gift codes and custom rooms. Stored in `prize_pools`, one row per day keyed by `dayKey`.

**Post**
A daily redeem-code post. Admin schedules ~5 per day. Each has a `scheduledAt`, ad config, and, once completed, awards 1 coin.

**Post completion**
Row in `post_completions`. One per user per post. Unique `{userId, postId}` blocks double-claim.

**PROGA**
Promotion and Regulation of Online Gaming Act 2025. India's national online gaming law. See `docs/OPEN_DECISIONS.md` item 1.

**Public / Pro / Pro Max**
The three subscription tiers. Public is free. Pro is ₹59/mo. Pro Max is ₹118/mo.

---

## R

**Razorpay**
India's dominant payment aggregator. Used for donations and subscriptions. Webhook is the source of truth.

**Redeem code**
See Gift code.

**Reveal time (`visibleFromAt`)**
Timestamp at which a custom-room's ID and password become visible to users. Default: 5 minutes before `scheduledAt`.

**Room ID / Room Password**
BGMI / FF lobby credentials. Encrypted at rest.

---

## S

**SAC code**
Services Accounting Code. 998439 for "Other on-line contents n.e.c." CashFB subscriptions use this code on GST invoices.

**Sponsor**
Brand logo on the home screen. Three slots, CMS-managed. Not the same as Top Donor (which is individual).

**Subscription payment**
Individual Razorpay charge on a subscription. Stored in `subscription_payments`. Has GST breakdown and invoice PDF URL.

---

## T

**TDS 194BA**
30% tax deducted at source on online game winnings. Applied to every `prize_pool_winners` row. Quarterly deposit via Challan 281. Form 26Q filing.

**Tier**
One of `PUBLIC`, `PRO`, `PRO_MAX`. User's current subscription level. Snapshot on `users.tier` with `tierExpiresAt`.

**Top donor**
The single highest cumulative donor. Surfaced on home screen. Materialised view in `top_donor_rankings` rank 1.

**Top 50 donors**
First 50 rows of `top_donor_rankings`. Refreshed every 5 minutes.

---

## V

**Vote**
Row in `votes`. One per user per day enforced by unique `{userId, dayKey}` index. Costs 3 coins. Contributes to tomorrow's prize pool.

---

## W

**Webhook event**
Row in `webhook_events`. Used for Razorpay webhook idempotency via unique `eventId` (`X-Razorpay-Event-Id` header).

**Worker service**
ECS Fargate service running `src/worker.ts`. Handles BullMQ jobs. No ALB target. Separate from `api-svc`.
